## Documentation Goals

The documentation should help a reviewer quickly understand:

- What the project does
- Why the project exists
- How the backend is structured
- Which endpoints are planned or available
- How data will be stored
- How to run and test the project
- What skills the project demonstrates

## Documentation Standards

Documentation should be:

- Clear
- Concise
- Accurate
- Updated as features change
- Written in plain language
- Useful to both developers and portfolio reviewers

## Maintenance Notes

When a new feature is added, update the relevant documentation file.

When a planned document is created, move it from the planned documents table into the current documents table.
